# COMP3011 Coursework 2 — Search Engine Tool

## Features

### Core Requirements

* Crawls the full website using Breadth-First Search with a 6-second politeness window
* Builds an inverted index storing frequency and position for every word on every page
* Saves and loads the index to and from disk
* Command-line shell with `build`, `load`, `print`, and `find` commands

### Advanced Features

* **TF-IDF Ranking** : results are ranked by relevance, not just matched. Words that appear often on a specific page but rarely across the site score highest
* **Phrase Search** : wrap a query in quotes to find pages where words appear consecutively, e.g. `find "be yourself"`
* **Query Suggestions** : if a word is not found, the engine automatically suggests the closest match using Python's built-in `difflib` library

## Design Decisions

### Data Structure — Inverted Index

The index is a nested dictionary:

python

```python
{
"love":{
"https://quotes.toscrape.com/":{
"frequency":2,
"positions":[194,268]
}
}
}
```

This was chosen because word lookup is O(1) using Python's hash table. Storing positions for every word was a deliberate decision ,without positions, phrase search would be impossible. Multi-word AND queries use set intersection across page sets.

### Crawler — Breadth-First Search

The crawler uses a queue to visit pages level by level, starting from the homepage. BFS ensures the most important pages (closest to the root) are crawled first. A 6-second sleep is enforced between every request. Network errors are caught gracefully so the crawler never crashes mid-crawl.

### TF-IDF Ranking

Results are ranked using TF-IDF:

```
TF  = word frequency on this page / total words on this page
IDF = log(total pages / pages containing this word)
Score = TF × IDF
```

Words that are frequent on one page but rare across the whole site score highest. Common words like "the" that appear everywhere score very low. This is the same principle real search engines use.

### Phrase Search

Phrase search checks whether words appear at consecutive positions in the index. For the query `"be yourself"`, the engine finds all pages containing both words, then checks whether `yourself` appears at exactly `position + 1` after every occurrence of `be`. Only pages with genuinely consecutive positions are returned.

### Query Suggestions

When a word is not found in the index, `difflib.get_close_matches` compares it against every word in the index using the Ratcliff/Obershelp similarity algorithm. If a word is at least 60% similar, it is suggested to the user. No extra libraries needed — difflib is part of the Python standard library.

## Testing

```bash
# Run all tests
python -m pytest tests/ -v

# Run with coverage report
pip install pytest-cov
python -m pytest tests/ --cov=src --cov-report=term-missing
```

The test suite contains **84 tests** across four files with **89% code coverage**, covering every module in the project.

- **test_crawler.py** — tests for page fetching, text extraction, link discovery, and the crawl loop. HTTP requests are simulated using `unittest.mock` so no real network connection is needed during testing.
- **test_indexer.py** — tests for tokenisation, index building, and saving and loading the index to disk using temporary files so the real index is never touched during testing.
- **test_search.py** — tests for TF-IDF scoring, phrase search, query suggestions, AND logic, and the full `cmd_find` function including all edge cases.
- **test_main.py** — tests for the CLI shell covering every command including `build`, `load`, `find`, `print`, `help`, and `quit`.

Edge cases tested include empty queries, unknown words, typos, numeric only input, case variations, phrase matching with non-consecutive positions, missing index file, empty quotes, and commands typed without arguments.
